### Changed

- Refactored `arc4.abi_call`, `arc4_create`, and `arc4_update` methods to use the new internal `ABICall` node, consolidating shared logic.

    **Potential Breaking Change:** Passing literal values when signature string does not specify type information is no longer accepted; e.g. `arc4.abi_call("foo", 256, app_id=1)` now throws `type information is needed when passing a literal value`
